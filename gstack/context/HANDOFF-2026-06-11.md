## Pegadinhas n8n 2.x (custaram iterações — não redescobrir)
1. `docker compose restart` NÃO recarrega `.env` — sempre `up -d`
2. Imagem sem `apk` e sem nó Execute Command — 7z via Dockerfile multi-stage +
   Code node com `execFileSync` (requer `NODE_FUNCTION_ALLOW_BUILTIN` no compose)
3. Runner sem `process` — usar `$env`; binário só via `this.helpers.getBinaryDataBuffer`
4. Sub-workflows exigem `n8n publish:workflow` + (webhooks) `--active=true` + restart
5. Nó Google Sheets: `authentication: "serviceAccount"` explícito; Update devolve SÓ a
   linha atualizada (dados extras → referenciar nó anterior via `$('Nome')`)
6. Cota de leitura do Sheets baixa → UM `values:batchGet` (HTTP Request + credencial
   googleApi com `httpNode:true` + scopes) em vez de N nós read; `retryOnFail` em tudo
7. `executeWorkflowTrigger` com `values: []` → "workflow has issues" (usar `passthrough`)
8. Telegram: botões URL (sendAndWait) exigem URL pública; teclados `callback_data`
   funcionam local (simuláveis por POST no webhook do roteador)
